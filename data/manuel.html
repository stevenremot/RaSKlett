RaSKlett – manuel d'utilisation

Introduction

RaSKlett est un logiciel pédagogique pour l’apprentissage de la théorie des combinateurs. Il fournit un langage de programmation fonctionnelle proche de cette théorie, et une interface graphique permettant de développer dans ce langage.

I) Le langage

Le langage RaSKlett permet de définir et d'utiliser des combinateurs.

Le nom du combinateur est défini par une chaîne de caractères. Par convention, il est recommandé de définir les combinateurs constants par une seule lettre majuscule.

Les combinateurs S, K et I sont natifs et ne peuvent pas être redéfinis.

L'expression := sert à la définition de combinateurs et de fonctions.

Le caractère ; est à mettre à la fin de chaque ligne de code.

Terminer une ligne de code par ;; permet de demander une réduction pas à pas de l’expression.

Exemple de définition :

J := S K K I ;

Une fonction prenant un ou plusieurs arguments peut être définie de plusieurs manières. Tout d'abord, n'oublions pas qu'une fonction est avant tout un combinateur sur lequel on exerce une abstraction.

L'abstraction de x dans une expression E s'écrit suivant l'algorithme λ+:

f := lambda+ x . E ;

On peut remplacer lambda+ par lambda++, lambda+++, lambda++++ suivant l'algorithme utilisé ou utiliser simplement lambda, auquel cas le compilateur choisira l'algorithme d'abstraction par défaut.

Pour une expression à plusieurs variables, il suffit de les énumérer avec des virgules :

f := lambda+ x,y . E(x,y) ;

Pour une fonction définie récursivement, il suffit d'énumérer le nom de la fonction à la suite des variables

f := lambda x,f . E(x,f) ;

Enfin, il est possible de définir une fonction implicitement, le système se chargeant de faire l'abstraction en utilisant l'algorithme par défaut.

f x := K (x + 1) S x K I ;

Sont définis en natif :

  1. Les booléens true et false, les opérateurs de booléens not : !, and : &&, or : ||
  2. Les entiers naturels, les opérateurs d'entiers : +, -, *, /
  3. Les opérateurs de test : =, !=, <, >, <=, >=
  4. La structure if .. then .. else

Exemple d'utilisation :

if <booleen>

then <expression>

else <expression> ;

  1. les listes [ <expression_head>, <expression>, … ]
  2. les fonctions head et tail (retournent la tête et la queue d’une liste)

L’utilisateur peut également ajouter des commentaires précédés du caractère #:

# a wild comment appears

Attention: ne pas lésiner sur les parenthèses, la précédence des opérateurs n’est pas toujours celle que vous pensez

II) L’interface

L'interface se présente sous la forme d'un unique champ de texte, à la manière d'un interpréteur, sur lequel l'utilisateur peut écrire des expressions basées sur le langage.

Il y a en particulier deux utilisations principales : écrire l'expression d'un combinateur pour obtenir sa réduction ou bien définir une fonction.

L’interface se compose ainsi de trois parties :

        - la barre d’outils, en haut, comprend différents boutons utiles pour les principales fonctionnalités du logiciel

        - la barre des combinateurs, à gauche, permet à l’utilisateur de voir tous les combinateurs définis nativement par le système, et de les activer ou de les désactiver

        - l’éditeur de code, au centre, sert à écrire les différentes expressions du langage et à afficher les résultats de la compilation et les éventuels rapports d’erreur

Compiler un code

Vous pouvez écrire un code, l’enregistrer ou en charger un de la même manière que dans tout autre logiciel.

Pour le compiler, vous avez plusieurs options visibles dans la barre d’outils et dans la barre de menus:

  1. Compiler tout le code: le code entier est compilé instruction par instruction, les résultats de chaque instruction s’affichant sous celle-ci
  2. Compiler la sélection: Après avoir sélectionné une partie du code, cette option la compile et affiche l’instruction au sein de laquelle se trouve le code sélectionné. Cette action peut être chaînée, en sélectionnant ensuite la partie du résultat à réduire.
  3. Compiler pas-à-pas: Lance une compilation contrôlée, où les choix est donné à tout instant d’effectue la réduction d’une étape, d’une instruction, ou du reste du code.

Une compilation trop longue peut être arrêtée avec l’action interrompre la compilation.

En cas d’erreur, le message d’erreur s’affiche là où aurait dû s’afficher le résultat de la compilation.

L’action nettoyer permet d’effacer tous les résultats et les messages d’erreurs affichés actuellement.

Attention: Le compilateur est réinitialisé à chaque lancement d’une nouvelle compilation. Aussi on ne peut compiler séparément la définition d’un combinateur et son utilisation.

Préférences

Les préférences, des Outils -> Préférences, permettent de configurer le programme.

Dans l’onglet Texte, il est possible de modifier la taille du texte dans l’éditeur, sa police, et d’afficher ou non les numéros de lignes.

L’onglet Compilateur permet de modifier le niveau d’abstraction par défaut du langage, lors d’expressions dans lesquelles le niveau d’abstraction est implicite:

f x := K x;

lambda x . K x;
